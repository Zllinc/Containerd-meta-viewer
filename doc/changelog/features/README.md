# 功能变更记录

本目录按功能分类记录 containerd-meta-viewer 的功能变更历史。

## 文档说明

每个功能都有一个独立的变更记录文件，记录该功能的所有历史变更。每次变更都要记录：
- **变更日期**（格式：YYYY-MM-DD）
- **变更前的实现方式**
- **变更后的实现方式**
- **变更原因和影响**

## 变更记录列表

- [数据库读取功能](database_reader.md) - 数据库读取器的变更历史
- [Buckets 命令](buckets_command.md) - buckets 命令的变更历史

## 如何记录变更

1. 找到对应的功能文件（如 `database_reader.md`）
2. 在文件顶部添加新的变更记录，格式：

```markdown
## YYYY-MM-DD: 变更标题

### 变更背景
说明为什么需要这个变更

### 之前的实现方式
描述变更前的实现

### 现在的实现方式
描述变更后的实现

### 变更原因
说明变更的原因和优势

### 影响范围
- 用户影响
- 性能影响
- 兼容性影响
```

3. 使用分隔线 `---` 分隔不同的变更记录

## 文档维护

根据开发文档要求，每次修改功能后，**必须**在此目录记录变更历史。
